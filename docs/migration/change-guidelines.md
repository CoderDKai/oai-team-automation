# 配置变更规范

## 目标

- 保证配置变更可追踪、可复核
- 降低跨模块变更的风险

## 影响分级

- **低影响**: 模块内实现调整，不改变对外接口与依赖关系
- **中影响**: 模块对外接口变更，但不影响依赖链路
- **高影响**: 影响跨模块依赖或引入新依赖，可能造成协作成本

## 评估维度

- 是否修改模块对外接口或契约
- 是否新增或调整依赖关系
- 是否影响多个模块的交付节奏

## 处理流程

- 低影响: 模块负责人自评后执行
- 中影响: 通知依赖模块负责人并同步评审
- 高影响: 组织联合评审并更新模块清单与文档

## 变更记录要求

- 更新对应的模块清单与入口文档
- 记录影响评估结论与复核结果
